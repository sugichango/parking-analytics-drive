# 🅿️ 駐車場データダッシュボード 完全構築ガイド

このドキュメントは、本システムの仕組み、構成、および将来的な再現（再デプロイ）のための手順をまとめた完全マニュアルです。

---

## 1. システムの全体像
本システムは以下の3つの要素が連携して動作しています。

1.  **ローカルPC (Windows)**: 日々のデータ更新作業を行う場所です。
2.  **GitHub (sugichango/parking)**: プログラムとデータの「保管庫」です。
3.  **Streamlit Community Cloud**: 全世界からアクセス可能な「Webアプリ本体」です。

---

## 2. フォルダ・ファイル構成
システムの安定動作のため、以下の構成になっています。

```text
parking_system/ (ルートフォルダ)
  │
  ├─ dashboard_app/             <-- 【重要】アプリの核となるフォルダ
  │   ├─ unified_dashboard.py   <-- 合体版プログラム本体
  │   ├─ requirements.txt       <-- 必要な部品リスト（pandas, openpyxlなど）
  │   ├─ README_admin.md        <-- 管理者マニュアル
  │   ├─ README_user.md         <-- 利用者マニュアル
  │   ├─ launch_dashboard.bat   <-- ローカル起動用スクリプト
  │   └─ data/                  <-- データフォルダ
  │       ├─ updated_integrated_data_FY2025.csv.gz (①用)
  │       └─ excel20XX_with_avg/ (②用の年度別フォルダ)
  │
  └─ .streamlit/
      └─ config.toml            <-- デザイン（ダークテーマ）設定
```

> [!IMPORTANT]
> **なぜ `dashboard_app` フォルダにまとめているのか？**
> Streamlit Cloudのサーバー（海外）では、日本語のフォルダ名（「駐車場データ分析」など）があると、データの読み込みや起動に失敗する深刻なバグがあるため、英語名のフォルダに集約しています。

## 3. 【再現手順】アプリを１から作り直す場合
もしPCを買い替えたり、設定が消えてしまったりした場合は、以下の手順で100%再現できます。

### 手順A：GitHubからデータを取得
1.  Gitをインストールし、リポジトリ（`sugichango/parking`）をクローン（ダウンロード）します。

### 手順B：Streamlit Cloudへのデプロイ
1.  [Streamlit Cloud](https://share.streamlit.io/) にログインします。
2.  **「Create app」** をクリックします。
3.  リポジトリに `sugichango/parking` を選択。
4.  **【最重要】Main file path** に以下を入力します。
    `dashboard_app/unified_dashboard.py`
5.  **「Advanced settings」** を開き、**Secrets** の欄に以下を貼り付けます（ログインパスワードの設定）。
    ```toml
    LOGIN_PASSWORD = "tutc_secure_login"
    ```
6.  **「Deploy!」** を押せば、数分でWeb上に公開されます。

## 4. データの更新方法
新しいデータを追加する場合は、以下の手順で行います。

### ① 経営ダッシュボード (Dashboard 1)
- `dashboard_app/data/updated_integrated_data_FY2025.csv.gz` を新しいファイルで上書きします。

### ② 稼働分析プロ (Dashboard 2)
1.  `dashboard_app/data/` 内に、`excel2026_with_avg` のような新年度フォルダを作成します。
2.  その中にExcelファイルを入れます。
3.  `unified_dashboard.py` の398行目付近にある `years = [2023, 2024, 2025]` に、`2026` を書き足します。

### ③ GitHubに送信（反映）
VS Codeのソース管理画面から、変更をコミットし **「Sync Changes（変更を同期）」** または **「Push」** を押してください。数分でクラウド版に自動反映されます。

## 5. 技術的な工夫（トラブル対応の歴史）
もし将来プログラムを書き換える際は、以下の2点に注意してください。

*   **絶対パスの回避**: プログラム内のパスは `BASE_DIR = os.path.dirname(os.path.abspath(__file__))` を使って、実行場所がどこであっても自動的に `data` フォルダを見つけられるように作られています。これにより「ファイルが見つかりません」エラーを防いでいます。
*   **openpyxl**: Excelを読み込むには `requirements.txt` に必ず `openpyxl` と記述されている必要があります。
