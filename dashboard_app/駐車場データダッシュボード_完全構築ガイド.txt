■ 🅿️ 駐車場データダッシュボード 完全構築ガイド (テキスト版)

このドキュメントは、本システムの仕組み、構成、および将来的な再現（再デプロイ）のための手順をまとめたマニュアルです。テキストエディタ（メモ帳など）で簡単に閲覧・編集できます。

--------------------------------------------------
1. システムの全体像
--------------------------------------------------
本システムは以下の3つの要素が連携して動作しています。

(1) ローカルPC (Windows): 日々のデータ更新作業（コピーやプッシュ）を行う場所です。
(2) GitHub (sugichango/parking): プログラムとデータの「保管庫（クラウド上の倉庫）」です。
(3) Streamlit Community Cloud: 全世界からアクセス可能な「Webアプリ本体（表示画面）」です。

--------------------------------------------------
2. フォルダ・ファイル構成
--------------------------------------------------
システムの安定動作と軽量化のため、以下の構成になっています。

[parking_system/] (ルートフォルダ)
  │
  └─ [dashboard_app/]           <-- 【重要】アプリの核となるフォルダ
       ├─ unified_dashboard.py   <-- 合体版プログラム本体 (UI/機能改善済み)
       ├─ requirements.txt       <-- 必要な部品リスト（設定ファイル）
       ├─ README_admin.md        <-- 管理者マニュアル (旧)
       ├─ README_user.txt        <-- 利用者マニュアル (テキスト版)
       └─ [data/]                <-- データフォルダ
           ├─ updated_integrated_data_FY2025.csv.gz (①一般利用分析データ)
           └─ excel20XX_with_avg/ (②24時間稼働分析用の年度別フォルダ)

3. 【最重要】職員の追加（招待）手順
--------------------------------------------------
職員を混乱させず、URLクリックだけで見れるようにするための確実な手順です。

(1) 管理画面にアクセス
    Streamlit Cloud (https://share.streamlit.io/) にサインインし、
    対象アプリの「Settings」>「Sharing」タブを開きます。

(2) メールアドレスの追加
    「Add viewers」に職員のアドレスを入力します。

(3) 【ここが重要！】通知メールを送らない
    入力欄の近くの **「Send notification（通知を送る）」のチェックを外して** から保存します。
    ※これにより、Streamlitからの「怪しいリンクだらけの英語メール」が届かなくなります。

(4) 杉田様からのメールで案内
    杉田様から作成済みの「お知らせメール（きれいなURL 1つ）」を送信します。
    ※URL: https://parking-yo789img2hvubb5jqzezgd.streamlit.app/
    職員はこのURLをクリックするだけで、本人確認を経てダッシュボードに入れます。

--------------------------------------------------
4. 【再現手順】アプリを１から作り直す場合
--------------------------------------------------
もし設定が完全に消えてしまった場合は、以下の手順で再現できます。

(1) 手順A：GitHubからデータを取得
    Gitをインストールし、リポジトリ（sugichango/parking）をダウンロードします。

(2) 手順B：Streamlit Cloudへのデプロイ
    1. 「Create app」をクリック。
    2. リポジトリに「sugichango/parking」を選択。
    3. Main file path に「dashboard_app/unified_dashboard.py」を入力。
    4. Advanced settings > Secrets にパスワードを設定（LOGIN_PASSWORD = ...）。
    5. 「Deploy!」で公開。

--------------------------------------------------
5. データの更新方法
--------------------------------------------------
(1) ①経営：gzファイルを上書き
(2) ②分析：新年度フォルダを作り、yearsリスト(399行目付近)を書き換える
(3) 反映：VS CodeでSync Changes(同期)を押す
