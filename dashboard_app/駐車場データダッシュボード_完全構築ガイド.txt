■ 🅿️ 駐車場データダッシュボード 完全構築ガイド (テキスト版)

このドキュメントは、本システムの仕組み、構成、および将来的な再現（再デプロイ）のための手順をまとめたマニュアルです。テキストエディタ（メモ帳など）で簡単に閲覧・編集できます。

--------------------------------------------------
1. システムの全体像
--------------------------------------------------
本システムは以下の3つの要素が連携して動作しています。

(1) ローカルPC (Windows): 日々のデータ更新作業（コピーやプッシュ）を行う場所です。
(2) GitHub (sugichango/parking): プログラムとデータの「保管庫（クラウド上の倉庫）」です。
(3) Streamlit Community Cloud: 全世界からアクセス可能な「Webアプリ本体（表示画面）」です。

--------------------------------------------------
2. フォルダ・ファイル構成
--------------------------------------------------
システムの安定動作と軽量化のため、以下の構成になっています。

[parking_system/] (ルートフォルダ)
  │
  └─ [dashboard_app/]           <-- 【重要】アプリの核となるフォルダ
       ├─ unified_dashboard.py   <-- 合体版プログラム本体 (UI改善済み)
       ├─ requirements.txt       <-- 必要な部品リスト（プログラムを動かすための設定）
       ├─ README_admin.md        <-- 管理者マニュアル
       ├─ README_user.md         <-- 利用者マニュアル
       └─ [data/]                <-- データフォルダ (最適化済み)
           ├─ updated_integrated_data_FY2025.csv.gz (①一般利用分析データ)
           └─ excel20XX_with_avg/ (②24時間稼働分析用の年度別フォルダ)

※注意：以前の「月ごとのバラのフォルダ（excel202304...等）」は今回の分析では使用しないため、動作高速化のためにすべて削除済みです。

--------------------------------------------------
3. 【再現手順】アプリを１から作り直す場合
--------------------------------------------------
もしPCを買い替えたり、設定が完全に消えてしまったりした場合は、以下の手順で再現できます。

(1) 手順A：GitHubからデータを取得
    Gitをインストールし、リポジトリ（sugichango/parking）をクローン（ダウンロード）します。

(2) 手順B：Streamlit Cloudへのデプロイ
    1. Streamlit Cloud (https://share.streamlit.io/) にGitHubアカウントでログインします。
    2. 「Create app」をクリックします。
    3. リポジトリに「sugichango/parking」を選択。
    4. 【最重要】Main file path に以下を入力します。
       dashboard_app/unified_dashboard.py
    5. 「Advanced settings」を開き、「Secrets」の欄に以下を貼り付けます（パスワード設定）。
       LOGIN_PASSWORD = "tutc_secure_login"
    6. 「Deploy!」を押せば、数分でWeb上に公開されます。

--------------------------------------------------
4. データの更新方法
--------------------------------------------------
新しいデータを追加する場合は、以下の手順で行います。

(1) ① 経営ダッシュボード (Dashboard 1)
    「dashboard_app/data/updated_integrated_data_FY2025.csv.gz」を新しいファイルで上書きします。

(2) ② 稼働分析プロ (Dashboard 2)
    1. 「dashboard_app/data/」内に、「excel2026_with_avg」のような新年度フォルダを作成。
    2. その中に各駐車場のExcelファイル（集計済み）を入れます。
    3. 「unified_dashboard.py」の399行目付近にある years = [2023, 2024, 2025] に「2026」を追加します。

(3) GitHubに送信（反映）
    VS Codeなどのツールで変更を確定（コミット）し、「Sync Changes（同期）」または「Push」を押すと、数分でクラウド版に反映されます。

--------------------------------------------------
5. 技術的な工夫（トラブル対応の歴史）
--------------------------------------------------
将来プログラムを書き換える際は、以下の点に注意してください。

(1) 絶対パスの回避
    プログラム内のパスは「プログラム自身の場所」を基準に自動生成されます。これによりクラウド上でもファイルが見つからないエラーを防いでいます。

(2) リソースの最適化
    不要な中間データや月別の重複データは削除し、GitHubの容量とクラウドの起動速度を最適化しています。

(3) openpyxlの指定
    Excelファイルを読み込むには「requirements.txt」に必ず「openpyxl」と記述されている必要があります。
